# 介绍

**Extenzo** 是基于 **Rsbuild** 的浏览器扩展开发框架，帮助你在同一套工程里开发、构建基于Chromium 与 Firefox 的浏览器扩展。Extenzo是**Extension** + **Enzo**的组合，Enzo是法拉利最经典的车型之一，也是法拉利创始人的名字，意味着超跑的典范。Extenzo旨在成为插件开发的超跑。

## 为什么用 Extenzo

浏览器扩展开发虽然仅使用基于 HTML+Javascript+CSS 的Web技术就能简单完成开发，但开发和调试的过程一直都没有得到很好的解决，包括热更新，错误监控与调试，前端框架集成等核心问题。之前社区出现了Plasmo和WXT这类优秀的框架，帮助开发者完成一站式的开发和体验。而Extenzo的区别是尽量减少对用户开发的约束，既保持极简的配置文件，又最大限度提供给用户自定义的可能。把更多浏览器插件开发的脏活封装起来，比如自动解析入口、热更新、浏览器自启动、自动安装插件等功能。这样带来的好处是开发者只需关注业务与 manifest 配置，对于复杂的大型项目更加可控。并且Extenzo带来了独一无二的错误监控面板，帮助开发者提高debug的效率。

## 与其它方案的区别

- **与纯手写 Webpack/Rspack/Rsbuild/Vite 相比**：Extenzo 封装了扩展特有的入口解析、manifest 注入、开发 HMR，你只需写业务与 manifest 配置。
- **与 Plasmo/WXT 等相比**：基于 Rsbuild，构建性能与生态与 Rsbuild 一致，尽量减少按特定的API和方式来进行代码编写，你可以按照你想要的方式完成开发。
