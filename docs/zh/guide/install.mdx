# 安装

## 方式一：脚手架创建项目

<Tabs>
  <Tab label="pnpm">

```bash
pnpm create extenzo-app
```

  </Tab>
  <Tab label="npm">

```bash
npx create-extenzo-app
```

  </Tab>
  <Tab label="yarn">

```bash
yarn create extenzo-app
```

  </Tab>
  <Tab label="bun">

```bash
bun create extenzo-app
```

  </Tab>
</Tabs>

可传入可选的项目名和选项：

```bash
create-extenzo-app [项目名] [选项]
```

- **项目名**：第一个参数（默认：`my-extension`）。若目录已存在，会提示是否覆盖。
- **选项**：
  - `--framework <name>` — 跳过选择：`vanilla` | `vue` | `react` | `preact` | `svelte` | `solid` | `uno`（React + UnoCSS）
  - `--language <lang>` — 跳过选择：`js` | `ts`
  - `--help` — 显示帮助
  - `--version` — 显示版本

按提示选择框架（Vanilla / Vue / React / Preact / Svelte / Solid / React+UnoCSS）和语言（JavaScript / TypeScript），会生成完整项目结构和配置。CLI 会自动识别包管理器，并在后续步骤中给出对应的安装与开发命令。

## 方式二：在现有项目中集成

### 1. 安装 extenzo

将 **extenzo** 作为**开发依赖**安装（一条依赖即包含 CLI 与构建能力）：

<Tabs>
  <Tab label="pnpm">

```bash
pnpm add -D extenzo
```

  </Tab>
  <Tab label="npm">

```bash
npm install -D extenzo
```

  </Tab>
  <Tab label="yarn">

```bash
yarn add -D extenzo
```

  </Tab>
  <Tab label="bun">

```bash
bun add -D extenzo
```

  </Tab>
</Tabs>

### 2. 添加配置文件

在项目根目录创建 `exo.config.ts`（或 `exo.config.js`），并确保项目中有 `background`、`content`、`popup`、`options`、`sidepanel` 等入口（在根目录或由 `appDir` 指定的目录下）。

### 3. 最小配置示例

```ts
// exo.config.ts
import { defineConfig } from "extenzo";
import { pluginReact } from "@rsbuild/plugin-react"; // 或 vue from "@extenzo/rsbuild-plugin-vue"

export default defineConfig({
  appDir: "src",
  outDir: "extension",
  manifest: {
    name: "My Extension",
    version: "1.0.0",
    manifest_version: 3,
    permissions: ["storage", "activeTab"],
    action: { default_popup: "popup/index.html" },
    background: { service_worker: "background/index.js" },
    content_scripts: [{ matches: ["<all_urls>"], js: ["content/index.js"] }],
  },
  plugins: [pluginReact()],
});
```

### 4. 包与导入

- **Core**：`defineConfig`、类型、入口发现与 manifest 等从 extenzo 导出。配置中请使用：`import { defineConfig } from "extenzo"`。
- **运行时**：需要 `browser` API 时请自行安装 [webextension-polyfill](https://github.com/mozilla/webextension-polyfill)，使用 `import browser from "webextension-polyfill"`。**Content UI**：`@extenzo/utils` 提供 `defineContentUI` / `mountContentUI`，用于在 content script 中注入 UI，使用 `import { defineContentUI, mountContentUI } from "@extenzo/utils"`。

### 5. 运行命令

- **开发**：`extenzo dev` 或使用包管理器的 run 脚本（如 package.json 中配置了 `"dev": "extenzo dev"`，则可用 `pnpm dev`、`npm run dev`、`yarn dev`、`bun run dev`）。
- **构建**：`extenzo build`，产物输出到 `outputRoot/outDir`（默认 `.extenzo/extension`）。

使用 `-l chrome|edge|brave|vivaldi|opera|santa|firefox` 指定目标浏览器，例如：

<Tabs>
  <Tab label="pnpm">

```bash

pnpm build
# 指定浏览器
pnpm dev -- -l chrome
pnpm build -- -l firefox
```

  </Tab>

  </Tab>
