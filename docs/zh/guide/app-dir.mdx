# 目录结构

Extenzo 的核心目录非常简单，主要分为四个部分：`.extenzo`、`app`、`public` 和 `exo.config.ts`：

```tree
.extenzo
app（可配置）
public
exo.config.ts
package.json
```

## .extenzo

`.extenzo` 目录由 Extenzo 自动生成，用于存放**构建产物**和**缓存**：

- 构建产物默认输出到 `.extenzo/extension`（可通过 [`outDir`](/config/out-dir) 修改目录名）。
- 缓存存放在 `.extenzo/cache`，用于加速后续构建并记录开发时的浏览器用户数据。

非必要情况下请不要手动删除 `cache` 目录，否则会影响构建速度以及浏览器用户配置的复用。

```tree
.extenzo/
├── extension   # 默认构建输出目录（outDir 可配置）
└── cache       # 构建与开发缓存
```

在开发模式下，浏览器实际加载的扩展目录也是 `.extenzo/extension`（或你自定义的 `outputRoot/outDir`），因此修改 `outDir` / `outputRoot` 时需同步检查 manifest 中的路径。

## exo.config.(js|ts)

`exo.config.ts` / `exo.config.js` 是 Extenzo 的**必需配置文件**，用于声明入口、manifest、输出目录等。详细字段请参考「配置」章节下各小节，例如：

- [`manifest`](/config/manifest)
- [`entry`](/config/entry)
- [`appDir`](/config/app-dir)

## app

Extenzo 默认以 `app` 目录作为所有入口和 `manifest` 文件的**应用根目录**。若需要修改，可在 `exo.config.ts` 中通过 `appDir` 字段指定：

```ts
export default defineConfig({
  appDir: "src",
});
```

`appDir` 用于指定**应用目录**，默认值为 `"app"`。入口发现（参见 [`entry`](/guide/entry/file-based.md)）和 manifest 自动加载（参见 [`manifest`](/config/manifest)）都基于 `appDir` 解析。框架会将其解析为绝对路径，例如 `appDir: "src"` → `path.resolve(root, "src")`。

## public

`public` 目录用于存放所有**静态资源**。这些资源会在构建时**原样复制**到输出目录中，不参与打包处理。例如：

```tree
public/
├── icons/
│   └── icon-128.png
├── _locales/
│   ├── en
│   └── ja
```

若使用默认配置（`outputRoot=".extenzo"`、`outDir="extension"`），构建后的目录结构大致为：

```tree
.extenzo/extension/
├── icons/
│   └── icon-128.png
├── _locales/
│   ├── en
│   └── ja
└── manifest.json
```

## 相关配置
