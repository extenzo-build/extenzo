# 国际化（i18n）

扩展的国际化资源（如多语言文案）在 Extenzo 项目中统一放在 **`public` 目录** 下。构建时 `public` 下的文件会被拷贝到输出目录（如 `dist`），因此 `_locales` 等目录结构会原样出现在产物中，供 Chrome 按 [default_locale](https://developer.chrome.com/docs/extensions/reference/manifest/default_locale) 与 `chrome.i18n` 规范加载。

常见做法是在项目根目录下建立 `public/_locales/<locale>/messages.json`（例如 `public/_locales/zh_CN/messages.json`），并在 manifest 中设置 `default_locale`。

具体消息格式、占位符用法及在 manifest/CSS/JS 中的引用方式，请参阅 Chrome 官方文档：

- [Internationalize your extension](https://developer.chrome.com/docs/extensions/develop/ui/internationalization)（国际化完整指南）
- [chrome.i18n API](https://developer.chrome.com/docs/extensions/reference/i18n/)（API 参考）
